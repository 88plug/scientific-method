# Impossibility brief
A vendor claims their "comparison-based sorting accelerator" sorts ANY
1,000,000 arbitrary comparable items in 1,000,000 comparisons worst-case
(i.e., n comparisons, n=1e6). Evaluate the claim with the impossibility
toolkit (invent.md §5): can ANY comparison sort do this? Produce the
strongest available verdict with the proof shape named, the arithmetic
shown, and the escape hatches stated (what relaxations WOULD make n
comparisons feasible).
